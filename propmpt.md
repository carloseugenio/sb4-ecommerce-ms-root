# Spring Boot 4 Ecommerce

# Propomt v1

I am starting a plan to create an Ecommerce application using Spring Boot 4.1 Cloud Services and 
Microservices using a monorepo approach with Gradle 9. Please read the repository from: 
[GitHub](https://github.com/carloseugenio/sb4-ecommerce-ms-root). The project layout is a proposal 
but not fixed and can be improved. The links on the README.md in the project, e.g.: 
[Microservices](https://levelup.gitconnected.com/stop-juggling-microservices-the-spring-boot-monorepo-blueprint-for-aws-6e8705e1d0f4), helps to clarify some decisions. 
The intent is that this project's cloud microservices can be deployed on AWS. For this, we need 
to use the components available in Spring Boot Cloud (API Gateway, Load balance, Resilience, etc.) 
and the AWS services, Docker and Kubernates. I want to evolve the model slowly as the requirements 
are comming in from Product. For now I only have this skeleton project.Make sure to use git bash for 
commands that need to be executed. Create the plan to evolve using an ADR in docs folder.
