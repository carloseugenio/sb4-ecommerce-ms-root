# Spring Boot 4 Ecommerce

# Propomt v1

I am starting a plan to create an Ecommerce application using Spring Boot 4.1 Cloud Services and 
Microservices using a monorepo approach with Gradle 9. The repository is on [GitHub](https://github.com/carloseugenio/sb4-ecommerce-ms-root). The project layout is a proposal but not fixed and can be improved. The links on the READM.md helps to clarify some decisions. The intent is that this project's cloud microservices can be deployed on AWS. For this, we need to use the components available in Spring Boot Cloud (API Gateway, Load balance, Resilience, etc) and the AWS services, Docker and Kubernates. I want to evolve the model slowly as the requirements are comming in from Product. For now I only have this skeleton project.Make sure to use git bash for commands that need to be executed. Create the plan to evolve using an ADR in docs folder.
